# Task

## Нужно разработать проект dota2_modding.
Это аналог moddota но с куда большим функционалом.

## Проект разделен на 4 ключевых дерриктории:
- data: это место хранение итоговых собранных и обработанных данных в формате json.

- api_dumper: программа для сбора и обработки данных собранных напрямую из игры путем подключения к netcon и получению http запросов от lua сервера.

- dota_addon: это аддон игры созданный через dota 2 workshop tools, который и будет запускаться в клиенте игры.

- dota2_modding: это итоговый сайт (аналог moddota), который будет публиковаться первое время на github pages. На нем через UI пользователи смогут изучать API игры. Все элементы будут в рантайме у клиентов собираться из json данных папки data.

## Список требуемого функционала dota2_modding:
1. Отображение всех vscript lua классов и их методов. А также C++ поля (они нужны для исследования и понимания, на практике в разработке кастомок скорее всего не приминимы). Отображение в виде древовидной структуры наследований (extends). Все методы не только lua сервера но и lua клиента. 
2. Отоюражение всех panorama классов и их методов.
3. Все типы панелей (Panel, Buttom, DOTAScenePanel ...).
4. Отображение всех css свойств и значений.
5. Отображение всех консольных команд.
6. Все game events, panorama events, все константы, энамы для vscripts и panorama. Списки всех существующих модификаторов, абилок, предметов, сущностей, все kv данные и их поля и значения. В целом все API и незадукоментированные возможности.
7. История изминений всего.
8. Глобальный поиск по всему сайту.
9. Отдельная страница с авторской документацией и гайдами на различные темы.

## Дополнительные детали
Нужно собирать не просто данные, а формировать из них целые объекты классы. Чтобы потом генерировать на основе всего этого lua и js аннотации.
